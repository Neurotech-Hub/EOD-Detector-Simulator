# MCP6561 Comparator Model

Microchip rail-to-rail comparator macromodel for ngspice, vendored at [`mcp6561.lib`](mcp6561.lib).

**Source:** Microchip MCP6561 SPICE model (Rev B, 09-Apr-2011)  
**Covers:** MCP6561, MCP6561R, MCP6561U, MCP6562, MCP6564

## Pin order

Subcircuit `MCP6561`:

| Pin | Name |
|-----|------|
| 1 | IN+ (non-inverting) |
| 2 | IN- (inverting) |
| 3 | VDD |
| 4 | VSS |
| 5 | OUT |

Example instance:

```spice
.include "models/mcp6561.lib"
Vsupply 3 0 5
Vref in_n 0 2.5
Vin in_p 0 PWL(0 1.0 1m 1.0 1.001m 3.0 5m 3.0)
XU1 in_p in_n 3 0 out MCP6561
```

## ngspice quirk: VDD node must be named `3`

This vendor model only switches correctly when the **VDD pin connects to a top-level net named `3`**. Using `VDD`, `vsup`, or other supply net names causes the output to stick near 0 V.

Workaround in your netlist:

```spice
Vsupply 3 0 5
XU1 in_p in_n 3 0 out MCP6561
```

Probe other nodes with descriptive names (`in_p`, `in_n`, `out`) as usual.

## ngspice status

Runs in ngspice 46 on transient analysis without file edits. Verified:

```bash
python scripts/run_stage.py --stage 00_sanity_mcp6561
```

Expected: Vout ≈ 0 V when IN+ < IN−, Vout ≈ 5 V when IN+ > IN−.

### Retired from the detector transient benches (stage 03)

This vendor macromodel is **no longer used in the stage 03 detector benches**. Its ESD clamp diodes (`D98`/`D99`, `DESD` with `IS=1e-15`) collapse the ngspice timestep **non-deterministically** when cascaded with the INA333 macromodel — identical runs pass or fail at random with "timestep too small" at `d.xu3.d98`. Exhaustive solver sweeps (`method=gear`/`trap`, `sparse`, `rshunt`, `itl4`, `cshunt`, `TMAX`, parasitic caps) either failed to fix this or distorted TRIGGER (e.g. `cshunt=1p` caused −0.8/+4.1 V rail overshoot).

Stage 03 now uses a **behavioral equivalent** ([`eod_comparator_behavioral.inc`](../stages/includes/eod_comparator_behavioral.inc)): threshold comparison against VTHRESH, rail-to-rail push-pull output referenced to the supply, and hysteresis from the physical R5/R9 feedback network — the electrically relevant behavior for EOD detection, without the stiff ESD network. The vendor includes (`eod_comparator.inc`, `eod_comparator_ti.inc`) remain in the tree, marked deprecated for transient use.

The macromodel is still exercised standalone in the `00_sanity_mcp6561` stage, where it converges fine.

The INA333 TI library may emit harmless `t_abs` warnings on internal resistor models; ngspice ignores them.

## Usage notes (from Microchip)

- Typical performance over −40 °C to +125 °C
- For high-impedance circuits, consider `.options gmin=100f`
- Hysteresis disabled in our vendored copy (`E12` uses `POLY(6)` no-hysteresis line) for reliable ngspice convergence

## Updating the model

1. Replace `mcp6561.lib` with a new vendor download
2. Re-apply header comment and hysteresis patch (see top of file)
3. Re-run the sanity stage and confirm the node-`3` VDD workaround still applies
